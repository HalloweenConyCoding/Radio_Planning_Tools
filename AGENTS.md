# AGENTS

## Design / Redesign Guidance
For UI design, redesign, styling, typography, color, motion, layout, accessibility, or anti-AI-slop work:
- Read `docs/design/impeccable/README.md` first.
- Then read only the relevant command files from `docs/design/impeccable/`.
- Preserve business logic, data logic, and API behavior unless the task explicitly asks otherwise.
- Do not redesign unrelated pages.
- Prefer targeted UI changes with clear before/after reasoning.
- After UI changes, run the project's normal build/check command if available.

## Project orientation files

Before making product, UI, layout, or design changes, read:

- `docs/product/PRODUCT.md`
- `docs/product/DESIGN.md`
- `docs/design/impeccable/` when working on visual design or styling
