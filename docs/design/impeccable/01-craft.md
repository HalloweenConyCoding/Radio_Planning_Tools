# /impeccable craft

Category: Create  
Source: https://impeccable.style/docs/craft/  
Note: Paraphrased study note based on the public Impeccable docs, not a verbatim copy.

## Purpose

Shape the design first, then build the feature in one end-to-end flow.

## Use when

- Starting a new feature from zero.
- You know what the feature should do, but the visual direction is not clear.
- You want discovery, implementation, and visual iteration managed as one workflow.

## How it works

- Runs a discovery interview about purpose, users, content, constraints, and desired direction.
- Loads relevant design references for spatial design, typography, motion, color, interaction, responsive behavior, and UX writing.
- Builds in order: structure, hierarchy, spacing, typography, color, states, motion, and responsive behavior.
- Checks the result visually and iterates instead of shipping the first working version.

## Example prompt

```text
/impeccable craft a pricing page for a developer tool
```

## Avoid / pitfalls

- Do not use it for small touch-ups.
- Do not rush the discovery interview.
- Do not skip visual iteration.
