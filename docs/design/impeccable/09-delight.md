# /impeccable delight

Category: Refine  
Source: https://impeccable.style/docs/delight/  
Note: Paraphrased study note based on the public Impeccable docs, not a verbatim copy.

## Purpose

Add small, brand-appropriate moments of personality to a solid interface.

## Use when

- The experience works but feels anonymous.
- You want memorable empty states, loading copy, success feedback, or microcopy.
- The core UI is already stable and polished.

## How it works

- Finds opportunities in empty states, loading/waiting moments, success feedback, microcopy, easter eggs, and secondary states.
- Reads brand tone from PRODUCT.md.
- Keeps delight optional: the interface must still work perfectly if the extra personality is removed.

## Example prompt

```text
/impeccable delight the first-run experience
```

## Avoid / pitfalls

- Do not force humor into serious products.
- Do not over-decorate.
- Run polish before delight; fix what is wrong before adding what is missing.

## Related commands

`/impeccable animate`, `/impeccable bolder`, `/impeccable onboard`, `/impeccable clarify`
