# /impeccable overdrive

Category: Refine  
Source: https://impeccable.style/docs/overdrive/  
Note: Paraphrased study note based on the public Impeccable docs, not a verbatim copy.

## Purpose

Create one extraordinary, high-ambition UI moment using advanced effects or interaction.

## Use when

- A specific moment needs to impress.
- The project budget allows technical complexity.
- You want WebGL, physics, cinematic transitions, canvas, GPU effects, or other uncommon UI techniques.

## How it works

- Chooses one moment to make extraordinary rather than spreading spectacle everywhere.
- May use shaders, spring physics, Scroll Timeline, View Transitions, canvas animation, or GPU filters.
- Budgets and profiles the result for smooth performance.
- Requires reduced-motion fallback.

## Example prompt

```text
/impeccable overdrive the landing hero
```

## Avoid / pitfalls

- Do not use it everywhere.
- Do not use it on reliability-first operator tools or dashboards.
- Do not ship slow spectacle; performance still matters.
- Do not run it before the base interface is solid.

## Related commands

`/impeccable animate`, `/impeccable delight`, `/impeccable optimize`, `/impeccable quieter`
