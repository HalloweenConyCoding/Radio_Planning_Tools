# /impeccable clarify

Category: Simplify  
Source: https://impeccable.style/docs/clarify/  
Note: Paraphrased study note based on the public Impeccable docs, not a verbatim copy.

## Purpose

Rewrite confusing functional UX copy so the interface explains itself.

## Use when

- Labels, buttons, errors, helper text, tooltips, or empty states confuse users.
- The problem is words, not layout or color.
- Users misunderstand fields, consequences, or next steps.

## How it works

- Improves labels and hints, button copy, error messages, empty states, tooltips/helper text, and confirmation dialogs.
- Uses audience and mental state from PRODUCT.md.
- Favors direct, outcome-based, non-blaming language.

## Example prompt

```text
/impeccable clarify the billing form
```

## Avoid / pitfalls

- Do not make copy cleverer when it needs to be clearer.
- Do not skip audience context.
- Do not use it for marketing copy; it is for functional UX text.

## Related commands

`/impeccable polish`, `/impeccable adapt`, `/impeccable onboard`, `/impeccable delight`
