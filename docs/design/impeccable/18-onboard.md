# /impeccable onboard

Category: Harden  
Source: https://impeccable.style/docs/onboard/  
Note: Paraphrased study note based on the public Impeccable docs, not a verbatim copy.

## Purpose

Design first-run experiences, empty states, and paths to initial user value.

## Use when

- New users do not know what to do first.
- Activation is weak.
- Empty states say nothing useful.
- Setup or onboarding blocks users before they reach value.

## How it works

- Defines the aha moment and designs the shortest path to it.
- Improves first-run screens, empty states, setup/installation, progressive disclosure, and activation feedback.
- Avoids both over-tutorialized onboarding and zero-onboarding blank screens.

## Example prompt

```text
/impeccable onboard the editor
```

## Avoid / pitfalls

- Do not default to product tours.
- Do not design onboarding before defining the aha moment.
- Do not run onboard on a broken core flow.

## Related commands

`/impeccable clarify`, `/impeccable delight`, `/impeccable shape`, `/impeccable harden`
